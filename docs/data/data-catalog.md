# 데이터 카탈로그 & 수집 계획

> 게임 구성에 필요한 **모든 데이터셋**의 목록, 스키마, 출처, 저작권 상태, 수집 방법.
> ai-env `project-data-env`의 raw→staging→marts 계층·재현성·품질 원칙을 따른다.

## 데이터 계층 (본 저장소 적용)
```
docs/data/raw-notes/     # 원천 조사 노트(불변 보존): 출처 URL·인용·근거
packages/engine/data/    # marts: 게임이 직접 소비하는 정제 JSON (검증됨)
scripts/data/            # staging→marts 변환·검증 스크립트(멱등)
```

## 필요한 데이터셋

| 데이터셋 | 내용 | 스키마 | 규모(원작) | 우선순위 |
|----------|------|--------|-----------|----------|
| **officers** | 장수 능력치·생몰 | `shared/Officer` | 수백 명 | P0 |
| **cities** | 41개 주 이름·인접·좌표 | `shared/City` | 41 | P0 |
| **scenarios** | 6개 시나리오 초기 배치 | `shared/Scenario` | 6 | P0 |
| **items** | 보검·보물·병법서(스탯 보정) | `shared/Item` | 수십 | P1 |
| **events** | 수명·재해·역사 이벤트 | `shared/GameEvent` | 다수 | P2 |
| **balance** | 명령 계수·전투/계략 확률식 | `engine/config` | — | P1 |

## 출처 & 저작권 분석 (중요)

| 자료 | 저작권 상태 | 본 프로젝트 방침 |
|------|-------------|-----------------|
| 게임 **메커니즘·규칙** (일기토·명령·계략) | 저작권 대상 아님(아이디어) | 자유 재현 ✅ |
| 실제 **삼국지 인물·지명·역사** | 공유 자산(역사) | 자유 이용 ✅ |
| 원작의 **정확한 능력치 수치 데이터셋** | KOEI 데이터/편집 저작물 소지 | **복제 금지.** 역사 기반 재산정 |
| 원작 **초상화·타일·UI·음악** | KOEI 저작물 | **사용 금지.** 오리지널/오픈 에셋 (→ `asset-strategy.md`) |
| 팬 위키(namu.wiki 등)의 정리 데이터 | 위키 라이선스 + 원작 파생 | 참고만. 그대로 복제 금지 |

> **원칙(SEC/거버넌스):** 타인의 저작물을 저장소에 그대로 담지 않는다.
> 데이터는 "역사적 사실 + 자체 밸런싱 산정"으로 재구성하고, 각 값의 근거를 `raw-notes`에 남긴다.

## 수집 제약 (현재 환경)
- 이 세션의 **egress 정책**이 namu.wiki·gamefaqs.com 등 상세 출처를 **차단(403)**한다.
  → 자동 스크레이핑 불가. WebSearch 요약 + 공개 역사 자료로 보완.
- 해소 방안(택1, 사용자 결정 필요):
  1. **역사 기반 자체 산정** — 위키피디아/정사·연의 공개 자료로 인물·지명 목록화 후 능력치 룰 산정(권장, 저작권 안전).
  2. **사용자 제공 데이터** — 사용자가 소유한 원작 자료/직접 정리한 표를 `raw-notes`에 투입.
  3. **egress 화이트리스트** — 관리자에게 특정 참고 도메인 허용 요청(참고용, 여전히 복제 아님).

## 수집·검증 파이프라인 (멱등)
```
raw-notes(근거) → scripts/data/build.ts (정규화+검증) → engine/data/*.json → 스키마 테스트
```
- 검증: JSON이 `shared` 타입/JSON Schema를 만족하는가, 능력치 1~100, 도시 인접 대칭,
  시나리오 참조 무결성(존재하는 officer/city id인지).
- 재현성: 같은 raw-notes → 같은 JSON. 변환은 결정론.

## 현재 시드 데이터 (provisional)
`packages/engine/data/`에 **소규모 검증용 시드**를 둔다. 실제 인물/지명이되,
능력치는 **placeholder(밸런싱 전)** 임을 각 파일에 명시. 스키마·로더·엔진을 증명하는 용도.
