# 명령 체계 분석

> 삼국지2는 한 도시에서 **장수 수만큼 명령**을 낼 수 있다(장수 1명 = 그달 행동 1회).
> 명령은 카테고리로 묶이며, 대부분 담당 장수의 능력치·금·군량을 소비한다.
> 아래는 구현 기준선 분류. 이식판별 명령 명칭/유무 차이는 데이터로 흡수.

## 카테고리별 명령 (기준선)

### 내정 (內政)
| 명령 | 효과 | 주 능력치 | 비용 |
|------|------|-----------|------|
| 개발 | 토지/개발도 상승 → 세수·수확 증가 | 지력 | 금 |
| 치수 | 치수 상승 → 수해 방지 | 지력 | 금 |
| 상업/시찰 | 민심·상업 진흥 | 매력 | 금 |
| 논밭(농업) | 곡창 확충 → 군량 수확 | 지력 | 금 |

### 군사 (軍事)
| 명령 | 효과 | 주 능력치 | 비용 |
|------|------|-----------|------|
| 징병 | 병사 모집(인구·매력 영향) | 매력 | 금 |
| 훈련 | 부대 훈련도↑(전투력) | 무력 | 금 |
| 무기/마 구입 | 장비 확충 | — | 금 |
| 군량 구입/방출 | 금↔군량 교환 | — | 금/쌀 |

### 인사 (人事)
| 명령 | 효과 |
|------|------|
| 등용 | 재야/타 세력 장수 영입(매력·상성·충성) |
| 포상 | 금·물자로 충성도↑ |
| 이동 | 장수·병력을 인접 주로 이동 |
| 탐색 | 재야 장수·아이템 발견 |

### 계략 (計略)  → 상세 `combat.md`
| 명령 | 효과 |
|------|------|
| 유언비어/선동 | 적 민심·충성 교란 |
| 이간 | 적 장수 충성도 저하(모반 유도) |
| 매수 | 적 장수 포섭 |

### 외교 (外交)
| 명령 | 효과 |
|------|------|
| 동맹 | 세력 간 불가침/연합 |
| 원조 | 금·군량 지원 |

### 전쟁 (戰爭)
| 명령 | 효과 |
|------|------|
| 침공 | 인접 적 주 공격 → 전투 페이즈(일기토·부대전) |

## 데이터 모델 (구현 계약)
명령은 엔진에서 **순수 함수**로: `apply(command, state) -> {state, events}`.
```ts
type CommandType =
  | 'develop' | 'floodControl' | 'commerce' | 'farm'
  | 'draft' | 'train' | 'buyArms' | 'trade'
  | 'recruit' | 'reward' | 'move' | 'search'
  | 'rumor' | 'sow' | 'bribe'
  | 'ally' | 'aid'
  | 'invade';

interface Command {
  type: CommandType;
  actorOfficerId: string;   // 명령 수행 장수
  cityId: string;
  params: Record<string, unknown>; // 대상 도시/장수/금액 등
}
```

## AI 에이전트와의 접점
- 엔진은 특정 상태에서 **합법적인(legal) 명령 목록**을 생성한다.
- AI 군주 에이전트는 그 합법 목록 안에서만 선택한다(규칙 위반 불가). → `docs/architecture/multi-agent.md`
