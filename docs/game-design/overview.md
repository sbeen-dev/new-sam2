# 삼국지2 (Romance of the Three Kingdoms II) — 원작 시스템 개요

> 목적: 웹 리뉴얼의 게임 규칙 기준선. 원작(1989, KOEI, PC-8801/9801, 이후 MS-DOS·SNES 등 이식)의
> 시스템을 정리한다. **수치·명령 상세는 이식판마다 차이**가 있으므로, 본 문서는 구현 기준선을 잡되
> 확정 수치는 원작 매뉴얼/공략으로 검증 후 `packages/engine/data/`에 반영한다.
>

## 장르 / 개요
- **턴제 전략 시뮬레이션.** 플레이어는 한 명의 군주(君主)가 되어 중국 대륙을 통일한다.
- 시리즈 2편. 소설 삼국지연의를 바탕으로 하되, 게임 시작 후 전개는 소설을 따르지 않는다.
- 한 턴 = **1개월**. 명령 → 각 세력 처리 → 이벤트/전투 → 다음 달.

## 승리 조건
- 모든 도시(주)를 하나의 세력이 지배하면 통일 → 게임 종료(승리).

## 삼국지2에서 새로 도입된 요소 (시리즈 맥락)
- **일기토(一騎討)** 최초 도입 — 전투 개시 시 1회. 거절 시 병력 손실. 무력 낮은 장수가 높은 장수를 이기면 그 차이만큼 무력 상승(예: 무력80이 100을 이기면 90으로).
- **군사(참모)로 지력 성장** — 참모보다 지력이 낮은 장수는 매달 지력 +1(참모 지력 −1까지).
- **계략(計略)** 시스템, **신군주(新君主, 커스텀 군주)** 생성.
- **포로 처리** — 등용/해방/참수. 이 작품에선 포로가 등용을 거절하지 못함.

## 핵심 루프 (턴 구조)
1. **명령 페이즈** — 각 도시에서 장수 수만큼 명령을 내린다(장수 1명 = 그달 행동 1회).
2. **세력 처리** — AI 세력(멀티에이전트)이 자기 도시에 명령을 내린다.
3. **해결 페이즈** — 전투·계략·이벤트·수입(세금/수확)·소비(군량) 정산.
4. **월 종료** — 다음 달로. 특정 월에 수확·재해·역병·인구 변동 등 계절 이벤트.

## 웹 리뉴얼 방향 (원작 대비)
- 규칙은 원작을 존중하되, **CPU 군주의 사고를 LLM 에이전트로 대체**해 매 플레이가 달라지도록 한다.
- 상세: `docs/architecture/multi-agent.md`.
