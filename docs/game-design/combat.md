# 전투·계략 시스템 분석

## 전투 흐름 (침공 시)

1. **일기토(一騎討) 제안** — 전투 개시 시 1회. 양측 장수가 1:1 대결.
   - 거절 가능하나 병력 손실 페널티.
   - 무력 차이가 승패 확률을 좌우(운 요소 포함).
   - **성장:** 무력 낮은 장수가 높은 장수를 이기면 그 차이만큼 무력 상승(예: 80이 100 이기면 90).
2. **부대 전투** — 병력·훈련·무력·지형으로 교전. 턴제로 병력 감소.
3. **결과** — 도시 함락(수비 병력 0/군주 후퇴) 또는 공격 실패(퇴각).
4. **포로 처리** — 사로잡은 적 장수: **등용 / 해방 / 참수**. (삼국지2에선 포로가 등용 거절 불가)

## 계략 (計略)

지력 기반의 비군사 공격. 성공 시 적을 약화, 실패 시 역효과/발각.

| 계략            | 효과                                                 |
| --------------- | ---------------------------------------------------- |
| 유언비어·선동   | 적 도시 민심 저하 → 폭동/이탈                        |
| 이간            | 적 장수 충성 저하 → 모반·이탈 유도                   |
| 매수            | 금으로 적 장수 포섭                                  |
| 화계/수계(전장) | 전투 중 지형·상황 이용한 특수 계략 _(이식판별 차이)_ |

성공 확률 ≈ f(시전자 지력, 대상 지력, 대상 충성/민심, 금 투입). 구현은 파라미터화.

## 아이템 (전리품)

전투 승리 후 획득:

- **보검류** → 장수 **무력** 상승
- **보물류** → 장수 **매력** 상승
- **병법서류** → 장수 **지력** 상승 _(이식판별)_

## 데이터 모델 (구현 계약)

```ts
interface DuelResult {
  winner: string;
  loser: string;
  warGain: number;
}
interface BattleResult {
  outcome: 'attackerTook' | 'attackerRepelled';
  attackerLosses: number;
  defenderLosses: number;
  captives: string[]; // 포로 officer id
}
```

전투는 **seed 주입 가능한 결정론 함수**로 구현(재현성·테스트 용이).
운 요소는 `Rng(seed)`로 주입 — 같은 seed·입력 → 같은 결과(→ 데이터 파이프라인 재현성 원칙).

## AI 에이전트 접점

- 전투 자체는 엔진(결정론). 에이전트는 **전투를 걸지/일기토를 받을지/포로를 어찌할지** 등
  전략적 선택만 담당.
