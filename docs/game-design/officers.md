# 장수(무장) 시스템 분석

> 원작 삼국지2의 장수 데이터 모델. 정확한 개별 수치는 KOEI 저작물이므로
> **본 저장소에는 역사적 인물을 기반으로 재구성한 데이터**를 둔다(→ `docs/data/data-catalog.md`).

## 능력치 (3대 스탯)

삼국지2는 장수를 **3개 능력치**로 표현한다(후속작의 통솔/정치 분리 이전). 범위 1~100.

| 스탯 | 원문                | 영향                                            |
| ---- | ------------------- | ----------------------------------------------- |
| 지력 | 智力 (Intelligence) | 계략 성공/방어, 내정(치수·개발) 효율, 참모 성장 |
| 무력 | 武力 (War)          | 일기토, 전투 시 부대 전투력                     |
| 매력 | 魅力 (Charm)        | 등용 성공률, 민심·충성 관리, 징병 효율          |

## 장수 부가 속성

| 속성       | 설명                                                            |
| ---------- | --------------------------------------------------------------- |
| 충성도     | 소속 군주에 대한 충성(1~100). 낮으면 타 세력 등용·모반 위험     |
| 소속       | 현재 섬기는 군주(무소속=재야)                                   |
| 소재 도시  | 현재 위치한 주(州)                                              |
| 신분       | 군주 / 일반 장수 / 재야 / 포로                                  |
| 연령·생몰  | 등장/사망 연도(시나리오·수명 이벤트에 사용)                     |
| 상성(相性) | 세력·인물 간 궁합값(등용·모반 확률 보정) _(이식판별 유무 차이)_ |

## 성장 메커니즘 (삼국지2 특징)

- **참모 성장:** 참모(군사)보다 지력이 낮은 장수는 매달 지력 +1 (참모 지력 −1 상한).
- **일기토 성장:** 무력 낮은 장수가 높은 장수를 이기면 그 차이만큼 무력 상승(80이 100을 이기면 90).
- **아이템 성장:** 전후 획득한 보검→무력, 보물→매력 상승(→ `docs/game-design/combat.md`).

## 데이터 모델 (구현 계약)

`packages/shared`의 `Officer` 타입으로 정의. 요약:

```ts
interface Officer {
  id: string; // 안정적 식별자 (예: "guan_yu")
  name: string; // 표시명 (관우)
  int: number; // 지력 1..100
  war: number; // 무력 1..100
  cha: number; // 매력 1..100
  bornYear: number;
  diedYear: number;
  // 런타임 상태(시나리오 로드 시 채워짐)
  lordId: string | null;
  cityId: string | null;
  loyalty: number; // 1..100
  status: 'lord' | 'officer' | 'free' | 'captive';
}
```

정적 데이터(능력치·생몰)와 런타임 상태(소속·위치·충성)를 **분리**한다.
정적은 `data/officers.json`, 런타임은 게임 세이브에 저장.
